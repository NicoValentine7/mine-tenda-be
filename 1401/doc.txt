# Docs

## in conclusion
完了報告はプロジェクトの終了を適切に記録し、関係者にその状態を通知する公式なプロセスの一部だと考えられます。
データ保存や特定条件に基づいた手続きが含まれており、プロジェクトの進捗状況を確認・報告するための重要なステップのようです。

	1.	対象の事業: 「補助事業 == 新築系 && ステータス == 交付決定」と条件に記載されているため、特定の新築プロジェクトに関する補助事業が対象です。この条件を満たしたプロジェクトが完了した際に報告が必要です。
	2.	データ保存: 完了報告のデータは「交付申請」と同様に保存されると記述されており、apply_numbersやapply_dataといったフィールドが関係しているようです。つまり、完了報告に関する情報は既存のデータ構造に従って記録されます。
	3.	関連資料: チケットや要求事項に基づいて実装されるような情報が前提として挙げられており、何らかの手続きや要件に従った形式で完了報告が行われることが示唆されています。

## data of 完了報告
apply_numbers, apply_dataあたりが関連している。
